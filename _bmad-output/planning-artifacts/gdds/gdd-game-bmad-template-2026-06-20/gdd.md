# 장기: 황산벌의 기억 — Game Design Document

**작성자:** GD
**게임 타입:** Turn-Based Tactics / Visual Novel Hybrid
**대상 플랫폼:** Web

---

## Executive Summary

### Core Concept

황산벌 전투(660년)의 두 영웅 — 계백 장군과 관창 화랑 — 의 시선을 통해 장기를 처음 만나는 문화적 몰입 경험. 규칙을 마스터하는 게임이 아니라, 이야기로 기억되는 게임.

POC는 계백의 시선으로 단일 챕터를 경험하며 장기의 핵심 기물 4개(차, 마, 포, 졸)를 역사 드라마 속에서 자연스럽게 체험한다.

### Target Audience

**1차 타겟:** K-컬처 팬 — 장기를 전혀 모르는 글로벌 사용자. K-드라마/K-팝을 통해 한국 문화에 관심을 가지고 전통 문화까지 탐구하고 싶은 층. 플레이 동기: 새로운 전략 게임이 아니라 한국 문화 경험의 연장.

**2차 타겟:** 장기를 배운 적 없는 한국 2030세대. "장기는 어른들 게임"이라는 인식 탓에 한 번도 접해본 적 없는 세대.

**POC 검증 대상:** 한국어 사용자로 핵심 가설 검증. 영어 현지화는 v1.0에서 추가.

### Unique Selling Points (USPs)

1. **스토리 기반 튜토리얼:** 규칙 설명이 아닌 역사 드라마로 장기를 소개하는 게임은 존재하지 않는다.
2. **아름다움이 경쟁력:** 수묵화 여백미와 김홍도 풍속화 스타일 비주얼이 기존 디지털 장기와의 결정적 차이를 만든다.
3. **감정을 담은 기물:** 같은 기물이 누구의 시선으로 보느냐에 따라 다른 감정으로 움직인다.
4. **글로벌 접근성 설계:** 외국인이 장기를 처음 접하는 최적의 진입점.

## Goals and Context

### Project Goals

**POC 핵심 가설 검증:**
> "수묵화 스타일의 역사 스토리텔링으로 장기를 처음 접한 사람을 몰입시킬 수 있는가?"

- 계백 시선 단일 챕터로 10~15분 경험 완성
- 완주율과 정성 피드백으로 핵심 가설 검증
- 2주 내 솔로 개발자가 구현 가능한 범위로 제한

### Background and Rationale

수천 년의 역사를 가진 장기는 디지털에서 제 모습을 찾지 못했다. 기존 디지털 장기 앱은 기능 중심에 머물고 UI는 구식이며, 비한국어권 사용자를 위한 설계가 전무하다. Chess.com 같은 세련된 포지셔닝의 디지털 장기는 아직 존재하지 않는다.

K-컬처의 글로벌 확산으로 전통 한국 문화 콘텐츠 수요가 상승하고 있다. 이 게임은 그 수요의 진입점을 설계한다.

## Core Gameplay

### Game Pillars

이 게임의 모든 설계 결정은 아래 3개 기둥에 답한다. 기둥에 위배되는 기능은 추가하지 않는다.

1. **문화적 몰입**
   플레이어는 이야기 속 장면을 통해서 장기를 문화로써 체험한다. 이야기의 흐름이 학습의 속도를 결정한다.
   → "기물 이동표를 보여줄까?" → NO. "계백이 차를 움직이며 설명할까?" → YES.

2. **시각적 아름다움**
   이 게임의 모든 시각적 요소는 여백미로 말한다.
   → 더하는 것이 아니라 덜어내는 것으로 표현한다.

3. **자연스러운 움직임**
   기물의 움직임은 그 기물을 바라보는 시선의 감정을 담는다.
   → 기물 종류가 아니라 시선이 감정을 결정한다. 계백의 시선 = 절실함/처절함. 관창의 시선 = 도전/패기.

**전제 (기둥이 아닌 바닥):** 장기의 전략성과 규칙은 수천 년의 역사가 증명한다. 별도로 설계하지 않는다.

### Core Gameplay Loop

POC 범위 (계백 챕터, 기물 3~4개):

```
역사 내레이션 (이야기 전진, 황산벌 전투 배경)
    ↓
기물 등장 — 전장의 인물로 소개
    ↓
이동 시연 — 기물이 어떻게 움직이는지 시각적으로 보여줌
    ↓
직접 움직여보기 (플레이어 액션)
    ↓
계백의 목소리 — 짧은 대사 (보상: 감정적 확인)
    ↓
다음 기물/장면으로 (동기: 이야기가 궁금하다)
    ↓
[마지막 기물 완료 시]
황산벌 전투 결말 장면
```

**잘못된 이동 처리:** 이동 불가 위치로의 이동은 조용히 막힌다. 이동 가능한 칸이 흐리게 표시되어 방향을 안내한다. 별도의 오류 피드백 없음.

### Win/Loss Conditions

POC에는 승/패 조건이 없다. **완주** 또는 **이탈**만 존재한다.

- **완주:** 모든 기물 체험 → 황산벌 전투 결말 장면 도달
- **이탈:** 중간 종료 (이탈 발생 지점이 POC 핵심 측정 지표)

## Game Mechanics

### Primary Mechanics

POC에서 소개하는 4개 기물. 각 기물은 장기 규칙을 따르되, 계백의 시선(절실함/처절함)으로 감정을 담아 표현한다.

| 기물 | 상징 | 이동 규칙 | 애니메이션 | 이징 | 사운드 |
|------|------|-----------|-----------|------|--------|
| 차(車) | 지휘하는 의지 | 가로/세로 직선, 거리 무제한, 기물에 막힘 | 500ms, 묵직하고 단호 | ease-in | 목재 미끄러지는 소리 |
| 마(馬) | 전령 | 직선 1칸 후 대각선 1칸 (L자), 인접 칸 막힘 | 300ms, 날렵하고 긴박 | ease-out | 가벼운 목재 타격음 |
| 포(砲) | 매복한 저격수 | 직선 이동, 반드시 기물 1개 넘어야 이동/공격, 포끼리 불가 | 150ms, 순간적이고 폭발적 | linear | 날카로운 타격음 |
| 졸(卒) | 백제의 마지막 병사들 | 전진 1칸 또는 횡보 1칸, 후퇴 불가 | 400ms, 비장하고 무겁게 | ease-in | 둔탁한 목재 소리 |

**잘못된 이동 처리:** 이동 불가 위치는 조용히 막힌다. 이동 가능한 칸이 흐리게 표시되어 방향을 안내한다. 별도 오류 피드백 없음.

**감정 원칙 (기둥 3):** 기물의 애니메이션 스펙은 기물 종류가 아니라 시선이 결정한다. v1.0 관창 챕터에서 동일한 기물이 다른 감정(도전/패기)으로 재정의된다.

### Controls and Input

- **기물 선택:** 기물 클릭 → 선택 상태 표시 + 이동 가능 칸 흐리게 표시
- **이동:** 이동 가능 칸 클릭 → 기물 이동 + 계백 대사 재생
- **선택 취소:** 다른 빈 칸 클릭 또는 동일 기물 재클릭
- **모바일:** 탭이 클릭과 동일하게 동작 (별도 처리 없음)
- **드래그 앤 드롭:** 미지원 (POC 범위 외)

## Turn-Based Tactics Specific Design

### Grid System and Movement

**보드:** 장기 표준 9×10 격자 (가로 9줄, 세로 10줄). SVG로 렌더링.

**기물 이동:** 장기 정식 규칙 적용. POC 소개 기물(차/마/포/졸) 이동 규칙:
- 차: 가로/세로 직선, 거리 무제한, 기물에 막힘
- 마: 직선 1칸 후 대각선 1칸, 인접 칸 막힘
- 포: 직선 이동, 기물 1개 반드시 넘어야 이동/공격, 포끼리 불가
- 졸: 전진 또는 횡보 1칸, 후퇴 불가

**이동 피드백:** 선택된 기물의 이동 가능 칸 흐리게 표시. 불가 이동은 조용히 막힘.

### Unit Types and Classes

POC에서 소개하는 4개 기물 → `## Game Mechanics` 참조.

전체 장기 기물(상, 사, 장/왕 포함)은 v1.0 이후 추가.

### Action Economy

POC는 자유 대국이 아닌 튜토리얼이다. 턴 구조 없음 — 플레이어가 원할 때 기물을 이동한다. 정해진 위치로 이동 시 다음 장면으로 자동 진행.

### Positioning and Tactics

POC 범위 외. v1.0 실전 대국 구현 시 설계.

### Terrain and Environmental Effects

장기 표준 보드 외 지형 효과 없음. 궁(宮) 영역은 사(士)·장(王) 이동 제한에 해당하나 POC 기물에는 미적용.

### Campaign Structure

**POC:** 단일 챕터 (계백 시선), 선형 진행, 6장면.

**로드맵:**
- v1.0: 관창 챕터 추가 → 양측 시선 완성
- v1.x: 온라인 멀티플레이
- v2.0: AI 대전

## Progression and Balance

### Player Progression

POC는 스킬 진행이 아닌 **내러티브 진행**이다. 플레이어는 기물을 하나씩 경험하며 장기에 대한 이해가 자연스럽게 쌓인다. 수치화된 진행 없음.

### Difficulty Curve

튜토리얼이므로 난이도 곡선 없음. 모든 기물 체험은 1회 성공으로 다음 장면으로 진행. 반복 시도 제한 없음.

### Economy and Resources

POC에는 자원/경제 시스템 없음.

## Level Design Framework

### Level Types

POC는 단일 챕터(계백 시선)로 구성된 선형 내러티브 튜토리얼이다. 레벨이 아닌 **장면(Scene)** 단위로 진행된다.

**장면 유형 2종:**
- **내러티브 장면:** 역사 내레이션 텍스트 + 배경 일러스트. 플레이어 입력 없음, 또는 "계속하기" 클릭.
- **체험 장면:** 기물 등장 소개 + 이동 시연 + 직접 움직여보기 + 계백 대사.

### Level Progression

POC 장면 순서 (총 6장면, 예상 플레이 시간 10~15분):

| 순서 | 장면 유형 | 내용 |
|------|-----------|------|
| 1 | 내러티브 | 오프닝 — 황산벌 전투 배경 설정, 계백의 결의 |
| 2 | 체험 | 차(車) — 지휘하는 의지 |
| 3 | 체험 | 마(馬) — 전령 |
| 4 | 체험 | 포(砲) — 매복한 저격수 |
| 5 | 체험 | 졸(卒) — 백제의 마지막 병사들 |
| 6 | 내러티브 | 결말 — 황산벌 전투 결말 장면 |

**장면/체험 비율:** 균형 (장면 1개 → 기물 체험 1개). 이야기와 학습이 번갈아 흐름.

## Art and Audio Direction

### Art Style

**스타일:** 수묵화 여백미 + 김홍도 풍속화 (인물 중심, 먹선+담채)

**원칙:** 덜어낼수록 아름답다 (기둥 2). 추가가 아니라 제거로 표현한다.

**컬러 팔레트:**
- 주조색: 먹색(黑), 한지색(米白)
- 포인트: 단청의 적색 또는 청색 (단 1색만 사용)

**UI:** 여백 중심, 군더더기 없는 레이아웃. 수묵화의 여백이 곧 UI의 미니멀리즘.

**제작 방식:** AI 생성 (Midjourney/Stable Diffusion) + GD 스타일 큐레이션. 스타일 프롬프트 표준화 및 시드(seed) 고정 필수 — 기물마다 화풍 불일치 방지.

**레퍼런스:**
- 김홍도 씨름: 인물 중심 구도, 배경 최소화, 먹선+담채, 움직임의 생동감
- Chess.com: 여백 중심 UI, 직관적 기물 조작 (미감은 제외)

### Audio and Music

**BGM:** 국악기 앰비언트 — 가야금, 해금, 대금. 톤: 절제된 고요함, 출전 전 결의의 분위기. 전투의 소란이 아니다.

**기물 이동 사운드 (4종):**
- 차(車): 목재 미끄러지는 소리 (묵직)
- 마(馬): 가벼운 목재 타격음 (경쾌)
- 포(砲): 날카로운 타격음 (폭발적)
- 졸(卒): 둔탁한 목재 소리 (비장)

**계백 대사:** 기물 이동 완료 시 짧은 대사 재생. POC에서는 텍스트 + 효과음으로 구현 (성우 녹음은 Out of Scope).

## Technical Specifications

### Performance Requirements

- 60fps 유지 (기물 이동 애니메이션 포함)
- 기물 이동 입력 후 애니메이션 시작까지 지연 없음 (< 16ms)
- 초기 로드 시간: [작성 중 — POC 완성 후 측정]

### Platform-Specific Details

- **플랫폼:** Web (브라우저)
- **지원 브라우저:** Chrome, Safari, Firefox 최신 버전
- **렌더링 방식:** SVG
- **애니메이션 구현:** CSS transitions / GSAP (기물 이동 스펙 구현)
- **모바일:** 탭 입력 지원 (별도 반응형 레이아웃은 v1.0 이후)

### Asset Requirements

- **보드:** SVG (장기판 9×10 격자, 수묵화 스타일 선)
- **기물:** SVG (차, 마, 포, 졸 — 김홍도 풍속화 스타일 인물)
- **배경 일러스트:** AI 생성 (Midjourney/Stable Diffusion) + GD 큐레이션, 황산벌 전투 장면
- **기물 캐릭터 일러스트:** AI 생성 + GD 큐레이션, 스타일 프롬프트 표준화 및 시드 고정 필수
- **내레이션 텍스트:** AI 생성 + GD 편집 (역사적 정확성 검토 필수)
- **사운드:** 목재 미끄러짐 / 가벼운 타격 / 날카로운 타격 / 둔탁한 타격 (4종), 국악기 앰비언트 BGM (가야금/해금/대금)

## Success Metrics

### Technical Metrics

| 지표 | 기준 | 측정 방법 |
|------|------|-----------|
| 기물 이동 정확도 | 4개 기물 모두 장기 룰에 맞게 이동, 오동작 0건 | 수동 테스트 (모든 이동 경우의 수 확인) |
| 프레임레이트 | 60fps 유지 (애니메이션 포함) | 브라우저 개발자 도구 |
| 이동 응답 지연 | 클릭 후 애니메이션 시작 < 16ms | 육안 확인 |

### Gameplay Metrics

| 지표 | 목표 | 측정 방법 |
|------|------|-----------|
| 완주율 | 결말 장면 도달 비율 측정 (목표치는 POC 후 설정) | 테스터 관찰 또는 간단한 이벤트 로깅 |
| 이탈 지점 | 이탈 발생 장면 번호 파악 | 테스터 세션 관찰 |
| 정성 피드백 | "아름답다" / "다시 해보고 싶다" / "장기를 더 배우고 싶다" 반응 확인 | 테스터 인터뷰 (5명 이상)

## Out of Scope

아래 항목은 POC 범위에서 명시적으로 제외한다. 의도적 생략이며 누락이 아니다.

**v1.0으로 이관:**
- 관창 시선 챕터 (도전/패기 감정선)

- AI 대전
