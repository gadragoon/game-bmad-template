# Story 1.7: 포(砲) 이동 룰 구현

Status: done

## Story

As a player,
I want to move the Po piece using its cannon jump rule — must jump over exactly one piece to move,
so that I experience the cannon's unique projectile-like movement pattern.

## Acceptance Criteria

1. **Given** the Po piece and a line with exactly 1 intervening piece (cannon platform)
   **When** `getValidMoves('po', pos, board)` is called
   **Then** all empty squares beyond that piece along the same line are returned as valid destinations

2. **Given** a line with 0 intervening pieces between Po and the board edge
   **When** valid moves are calculated for the Po
   **Then** no destinations in that direction are returned (cannot move without jumping)

3. **Given** a line where a second piece appears immediately after the cannon platform
   **When** valid moves are calculated
   **Then** no destinations beyond the second piece are returned in that direction

4. **Given** a destination square occupied by any piece (including another Po)
   **When** valid moves are calculated
   **Then** that square is excluded — Po cannot capture in single-team POC, and Po cannot capture a Po (포끼리 포획 불가) in any case

5. **Given** an intervening piece that is a Po (ally cannon)
   **When** valid moves are calculated
   **Then** the jump is permitted — Po can use another Po as cannon platform

6. **Given** `pieces/rules/po.ts`
   **When** the file is inspected
   **Then** it has zero React or DOM imports — pure function only

### 권장 구현 코드

```typescript
import type { Position, Board } from '../../types'
import { BOARD } from '../../constants'

const DIRECTIONS = [
  { dr: -1, dc:  0 }, // 상
  { dr: +1, dc:  0 }, // 하
  { dr:  0, dc: -1 }, // 좌
  { dr:  0, dc: +1 }, // 우
] as const

export function getPoValidMoves(pos: Position, board: Board): Position[] {
  const result: Position[] = []

  for (const { dr, dc } of DIRECTIONS) {
    // Phase 1: cannon platform 탐색
    let r = pos.row + dr
    let c = pos.col + dc
    let platformFound = false

    while (r >= 0 && r < BOARD.rows && c >= 0 && c < BOARD.cols) {
      if (board[r][c] !== null) {
        platformFound = true
        r += dr
        c += dc
        break
      }
      r += dr
      c += dc
    }

    if (!platformFound) continue

    // Phase 2: 경유 기물 이후 목적지 수집
    while (r >= 0 && r < BOARD.rows && c >= 0 && c < BOARD.cols) {
      if (board[r][c] !== null) break  // 단일 팀 POC + 포끼리 포획 불가
      result.push({ row: r, col: c })
      r += dr
      c += dc
    }
  }

  return result
}
```

**알고리즘 특성 (AC4, AC5 관련):**
- Phase 1에서 기물 타입 검사 없음 → 어떤 기물이든(Po 포함) platform 가능 (AC5 자동 충족)
- Phase 2에서 기물 타입 검사 없음 → 모든 occupied 칸 제외 → 포끼리 포획 불가 자동 충족 (AC4)
