# Story 2.2: 씬 1 — 오프닝 내러티브 장면

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a player,
I want to read the opening narration about the Battle of Hwangsanbul with a background illustration and a "계속하기" button,
so that I'm immersed in the historical context and Gyebaek's resolve before gameplay begins.

## Acceptance Criteria

1. **Given** `state.scene === 'opening'`
   **When** the Opening 컴포넌트가 렌더됨
   **Then** 황산벌 전장을 나타내는 배경 일러스트(플레이스홀더)가 표시됨

2. **Given** 오프닝 씬
   **When** 일러스트와 텍스트 콘텐츠를 확인
   **Then** 계백의 결의와 660년 역사적 배경을 전달하는 내레이션 텍스트가 표시됨

3. **Given** 오프닝 씬이 완전히 렌더된 상태
   **When** 플레이어가 "계속하기"를 클릭 또는 키보드로 트리거
   **Then** `dispatch({ type: 'NEXT_SCENE' })`가 호출되고 `state.scene === 'cha'`로 전환됨

4. **Given** 배경 일러스트 에셋
   **When** 스타일 프롬프트 메타데이터를 확인
   **Then** 수묵화 스타일 프롬프트와 시드(seed) 값이 코드 내에 문서화되어 있음 (NFR5 — AI 일러스트 화풍 일관성, Story 2.7 결말 씬에서 동일 표준 재사용 예정)

5. **Given** 오프닝 씬 텍스트
   **When** 한국어 사용자가 텍스트를 읽음
   **Then** 내레이션은 황산벌 전투에 대해 역사적으로 정확해야 함 (660년, 계백 장군, 5천 결사대, 나당연합군 — 수동 콘텐츠 검수 대상, 자동 테스트 불가)

## Dev Notes

### 현재 구현 상태 — Opening.tsx (Story 2.1에서 생성된 스텁, 지금 UPDATE 대상)

```tsx
// src/scenes/Opening.tsx (현재)
import { useGameDispatch } from '../state/GameContext'
import styles from './Scene.module.css'

export default function Opening() {
  const dispatch = useGameDispatch()
  return (
    <div data-scene="opening" className={styles.scene}>
      <p>[황산벌 오프닝 내레이션 자리 — Story 2.2에서 작성]</p>
      <button onClick={() => dispatch({ type: 'NEXT_SCENE' })}>계속하기</button>
    </div>
  )
}
```

`data-scene="opening"` 및 "계속하기" 버튼(`dispatch({ type: 'NEXT_SCENE' })`)은 **이미 완전히 동작함** — Story 2.1의 Playwright 테스트(`tests/story-2-1-scene-transition.spec.ts`)가 이 버튼에 의존하므로 **버튼의 텍스트("계속하기")와 클릭 동작은 절대 변경하지 말 것**. `<p>` 플레이스홀더만 교체 대상.

### 이미지 생성 도구 미연동 — 플레이스홀더 처리 방침

GDD는 배경 일러스트를 "AI 생성(Midjourney/Stable Diffusion) + GD 큐레이션"으로 명시하지만, 현재 프로젝트에는 이미지 생성 MCP나 자동화 파이프라인이 연동되어 있지 않다. 이 스토리에서는:
- 실제 이미지 파일을 생성하지 않는다 (범위 외 — GD가 별도로 처리)
- `--color-hanji`/`--color-sumi` CSS 변수(`Board.module.css`에 이미 정의됨, `:root`로 전역 방출됨)를 사용한 그라디언트 placeholder로 "배경 일러스트 자리"를 시각적으로 표현한다
- `data-style-prompt`/`data-style-seed` 속성으로 향후 실제 이미지 교체 시 참조할 메타데이터를 코드에 문서화한다 (AC4가 요구하는 것은 실제 이미지가 아니라 **문서화된 메타데이터**)

### narration.ts 설계 — Story 2.7과의 재사용 계약

`SUMI_E_STYLE`은 Story 2.7(결말 씬)의 AC에서 "Story 2.2에서 문서화한 표준과 일치"할 것을 요구한다. 따라서 이름을 `OPENING_STYLE`이 아닌 범용 `SUMI_E_STYLE`로 정의하고, Ending.tsx가 동일 상수를 import하도록 설계 의도를 남겨야 한다.

```typescript
// src/story/narration.ts (신규 — 예시 구조, 실제 텍스트는 역사적 정확성 검수 완료 상태로 작성)
export const SUMI_E_STYLE = {
  prompt: 'sumi-e ink wash painting, Korean historical battlefield, minimal negative space, muted ink black and hanji cream palette, single dancheong accent',
  seed: 660660, // 660년 역사적 배경을 상징하는 고정 시드 — 기물/배경 전체 동일 시드 사용
} as const

export const OPENING_NARRATION = {
  title: '황산벌, 660년',
  paragraphs: [
    '660년, 나당연합군이 백제를 향해 진격한다.',
    '계백은 5천 결사대와 함께 황산벌에 최후의 방어선을 세운다.',
    '살아 돌아갈 길은 없다. 오직 지켜야 할 것이 있을 뿐이다.',
  ],
} as const
```

### Opening.tsx 변경 패턴

```tsx
import { useGameDispatch } from '../state/GameContext'
import { OPENING_NARRATION, SUMI_E_STYLE } from '../story/narration'
import styles from './Scene.module.css'

export default function Opening() {
  const dispatch = useGameDispatch()
  return (
    <div data-scene="opening" className={styles.scene}>
      <div
        data-scene-bg
        role="img"
        aria-label="황산벌 전장 배경"
        data-style-prompt={SUMI_E_STYLE.prompt}
        data-style-seed={SUMI_E_STYLE.seed}
        className={styles.sceneBackground}
      />
      <h1>{OPENING_NARRATION.title}</h1>
      {OPENING_NARRATION.paragraphs.map((p) => (
        <p key={p} className={styles.narrationText}>{p}</p>
      ))}
      <button onClick={() => dispatch({ type: 'NEXT_SCENE' })}>계속하기</button>
    </div>
  )
}
```
