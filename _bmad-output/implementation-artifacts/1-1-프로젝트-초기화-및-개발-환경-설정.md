# Story 1.1: 프로젝트 초기화 및 개발 환경 설정

Status: done

## Story

As a developer,
I want a fully configured project environment with all dependencies installed and folder structure in place,
so that I can immediately begin implementing game features without setup friction.

## Acceptance Criteria

1. **Given** 터미널이 프로젝트 디렉토리에서 열려 있을 때  
   **When** `npm run dev` 실행  
   **Then** Vite dev 서버가 시작되고 localhost에서 에러 없이 앱에 접근 가능

2. **Given** `src/assets/` 안에 SVG 파일이 있을 때  
   **When** `import Piece from './piece.svg?react'` 로 import  
   **Then** TypeScript 컴파일 에러 없이 React 컴포넌트로 렌더링됨

3. **Given** `src/audio/audioManager.ts` 파일이 있을 때  
   **When** 파일 내용 검사  
   **Then** 이 파일만 Howler.js를 직접 import — 다른 모든 파일은 `audioManager.play(key)` 만 사용

4. **Given** `vite.config.ts` 설정 상태에서  
   **When** `npm run build` 실행  
   **Then** `/janggi-hwangsanbul/` base path로 `dist/` 출력 생성 (GitHub Pages 호환)

5. **Given** `src/` 디렉토리 탐색 시  
   **When** 폴더 구조 확인  
   **Then** 다음 폴더가 존재: `scenes/`, `board/`, `pieces/rules/`, `pieces/config/`, `story/`, `audio/`, `state/`, `assets/`

### 프로젝트 초기화 명령어 (순서대로 실행)

```bash
# 1. 프로젝트 생성 (폴더 이름은 반드시 janggi-hwangsanbul — GitHub Pages base path와 일치)
npm create vite@latest janggi-hwangsanbul -- --template react-ts
cd janggi-hwangsanbul

# 2. 기본 의존성
npm install

# 3. 게임 의존성
npm install gsap @gsap/react
npm install howler
npm install -D vite-plugin-svgr @types/howler gh-pages

# 4. 실행 확인
npm run dev
```

2. `state/` → `scenes/`, `board/`, `pieces/` 단방향 의존
3. `constants.ts` — GDD 명세 숫자값 유일한 위치

### Project Structure Notes

- 프로젝트 루트 폴더명: `janggi-hwangsanbul` — GitHub Pages base path `/janggi-hwangsanbul/`와 일치 (변경 금지)
- `vite.config.ts`의 `base` 값은 배포 후 에셋 경로에 직접 영향 — 잘못 설정 시 모든 에셋 404
- `vite-plugin-svgr` 없이는 `import X from '*.svg?react'` 구문 컴파일 에러 발생
- **`vite-env.d.ts`에 `/// <reference types="vite-plugin-svgr/client" />` 추가 필수** — 없으면 TS2307 에러

### Project Context Rules

- **GSAP 사용 시**: `useGSAP({ scope: ref })` + `contextSafe()` 래퍼 필수. `gsap.to()` 직접 호출 금지 (메모리 누수)
- **오디오**: 어떤 파일에서도 `import { Howl } from 'howler'` 금지. 반드시 `audioManager.play(key)` 사용
- **룰 엔진** (`pieces/rules/`): React, DOM import 절대 금지 — 순수 함수만
- **상태 변경**: `dispatch`만 사용 — `useState` 직접 설정 금지 (Story 1.2 이후)
- **매직 넘버 금지**: 500, 300, 150, 400 (애니메이션 ms) 등은 반드시 `constants.ts`에서만 참조
- **에러 노출 금지**: 예외를 플레이어 UI에 절대 표시하지 않음
- **개발 로그**: `if (import.meta.env.DEV) console.log('[태그]', ...)` 패턴 사용
