# Story 1.5: 기물 선택·이동 인터랙션 및 차(車) 룰 구현

Status: done

## Story

As a player,
I want to click a piece to select it, see where it can move, and click a valid square to move it there,
so that I can play the game using the Cha piece with correct janggi rules.

## Acceptance Criteria

1. **Given** pieces are rendered on the board
   **When** a player clicks the Cha piece
   **Then** the Cha is visually highlighted as selected and all valid destination squares are dimly highlighted

2. **Given** the Cha is selected and valid squares are highlighted
   **When** a player clicks a highlighted valid square
   **Then** the Cha moves to that square and the selection/highlights clear

3. **Given** the Cha is selected
   **When** a player clicks the Cha piece again
   **Then** selection is cancelled and all highlights disappear — no error is shown

4. **Given** the Cha is selected
   **When** a player clicks an empty non-valid square
   **Then** selection is cancelled silently — no error message, alert, or console error appears (FR10)

5. **Given** the Cha at any position
   **When** `getValidMoves('cha', pos, board)` is called
   **Then** it returns all reachable squares along horizontal and vertical lines, stopping before (not including) the first blocking piece in each direction

6. **Given** `pieces/rules/cha.ts`
   **When** the file is inspected
   **Then** it has zero React or DOM imports — it is a pure function with no side effects

### getChaValidMoves 구현 패턴

```typescript
// src/pieces/rules/cha.ts
import type { Position, Board } from '../../types'
import { BOARD } from '../../constants'

const DIRECTIONS = [
  { dr: 0, dc: 1 },   // 우
  { dr: 0, dc: -1 },  // 좌
  { dr: 1, dc: 0 },   // 하
  { dr: -1, dc: 0 },  // 상
] as const

export function getChaValidMoves(pos: Position, board: Board): Position[] {
  const result: Position[] = []
  for (const { dr, dc } of DIRECTIONS) {
    let r = pos.row + dr
    let c = pos.col + dc
    while (r >= 0 && r < BOARD.rows && c >= 0 && c < BOARD.cols) {
      if (board[r][c] !== null) break  // 기물 만남 → 이 칸 포함 안 함, 루프 종료
      result.push({ row: r, col: c })
      r += dr
      c += dc
    }
  }
  return result
}
```

> **단일 팀 시나리오:** 기물 칸을 포함하지 않는다 (포획 없음). AC 5 명시: "stopping before (not including) the first blocking piece".

> **board 인덱싱:** 반드시 `board[row][col]` 순서. `board[col][row]` 실수 금지.

### HintOverlay 구현 패턴

```typescript
// src/board/HintOverlay.tsx
import { useMemo } from 'react'
import { useGameState } from '../state/GameContext'
import { useGameDispatch } from '../state/GameContext'
import { getValidMoves } from '../pieces/rules/ruleEngine'
import { positionToCoords, CELL_SIZE, BOARD_PADDING } from './gridUtils'
import { BOARD } from '../constants'
import styles from './HintOverlay.module.css'

const VIEWBOX_W = (BOARD.cols - 1) * CELL_SIZE + 2 * BOARD_PADDING
const VIEWBOX_H = (BOARD.rows - 1) * CELL_SIZE + 2 * BOARD_PADDING
const HINT_RADIUS = 22  // CELL_SIZE(60) 대비 약 37% — 기물보다 작게, 명확하게 인식 가능

export default function HintOverlay() {
  const { selectedPiece, board } = useGameState()
  const dispatch = useGameDispatch()

  const validMoves = useMemo(() => {
    if (!selectedPiece) return []
    for (let r = 0; r < BOARD.rows; r++) {
      for (let c = 0; c < BOARD.cols; c++) {
        if (board[r][c] === selectedPiece) {
          return getValidMoves(selectedPiece, { row: r, col: c }, board)
        }
      }
    }
    return []
  }, [selectedPiece, board])

  if (!selectedPiece) return null

  return (
    <svg
      className={styles.hintOverlay}
      viewBox={`0 0 ${VIEWBOX_W} ${VIEWBOX_H}`}
      preserveAspectRatio="xMidYMid meet"
      aria-hidden="true"
    >
      {validMoves.map((pos) => {
        const { x, y } = positionToCoords(pos)
        return (
          <circle
            key={`${pos.row}-${pos.col}`}
            cx={x} cy={y} r={HINT_RADIUS}
            className={styles.hintCircle}
            onClick={(e) => {
              e.stopPropagation()
              dispatch({ type: 'MOVE_PIECE', to: pos })
            }}
            tabIndex={0}
            onKeyDown={(e) => {
              if (e.key === 'Enter' || e.key === ' ') {
                e.preventDefault()
                dispatch({ type: 'MOVE_PIECE', to: pos })
              }
            }}
          />
        )
      })}
    </svg>
  )
}
```

### Piece.tsx 업데이트 패턴

```tsx
// src/pieces/Piece.tsx — 변경 부분만 표시
type Props = {
  pieceId: PieceId
  position: Position
  config: PieceConfig
  isSelected: boolean   // ← 신규
}

function Piece({ pieceId, position, isSelected }: Props) {
  const pieceRef = useRef<SVGGElement>(null)
  const dispatch = useGameDispatch()    // ← 신규
  const { x, y } = positionToCoords(position)
  const SvgIcon = SVG_MAP[pieceId]

  const handleClick = (e: React.MouseEvent | React.KeyboardEvent) => {
    e.stopPropagation()  // 배경 rect DESELECT 이벤트 차단 — 필수!
    if (isSelected) {
      dispatch({ type: 'DESELECT_PIECE' })
    } else {
      dispatch({ type: 'SELECT_PIECE', id: pieceId })
    }
  }

  return (
    <g
      ref={pieceRef}
      className={`${styles.piece}${isSelected ? ` ${styles.selected}` : ''}`}
      transform={`translate(${x}, ${y})`}
      onClick={handleClick}
      role="button"
      tabIndex={0}
      aria-pressed={isSelected}
      aria-label={`기물 ${pieceId}`}
      onKeyDown={(e) => {
        if (e.key === 'Enter' || e.key === ' ') {
          e.preventDefault()
          handleClick(e)
        }
      }}
    >
      <SvgIcon x={-30} y={-30} width={60} height={60} />
    </g>
  )
}
```

### 이벤트 전파 흐름 (핵심!)

클릭 이벤트가 어떻게 처리되는지 반드시 이해:

```
1. 기물 클릭:
   Piece.onClick → e.stopPropagation() → SELECT/DESELECT dispatch
   (이벤트 PiecesLayer 배경 rect에 도달 안 함)

2. 힌트 원 클릭:
   circle.onClick → e.stopPropagation() → MOVE_PIECE dispatch
   (이벤트 PiecesLayer 배경 rect에 도달 안 함)

3. 빈 영역 클릭:
   PiecesLayer 배경 rect.onClick → DESELECT_PIECE dispatch (selectedPiece 있을 때만)

4. 다른 기물 클릭 (선택 중):
   Piece.onClick → e.stopPropagation() → SELECT_PIECE(다른 기물) dispatch
   (자동으로 이전 선택 교체 — gameReducer.ts SELECT_PIECE 케이스가 selectedPiece 덮어씀)
```

### Project Context Rules

**React + GSAP 통합 (이번 스토리: GSAP 미사용)**
- `useGSAP + contextSafe` — 이번 스토리에서 GSAP 없음. pieceRef는 Story 3.1용으로 유지
- `will-change: transform` 기존 유지

**장기 룰 엔진**
- `getValidMoves(pieceId, pos, board)` 단일 진입점 — Story 1.5에서 HintOverlay가 사용
- `getChaValidMoves`는 순수 함수 — React/DOM import 절대 금지
- 기물별 함수 직접 호출 금지 — `ruleEngine.ts`의 `getValidMoves` 통해서만

**씬 상태 머신**
- `dispatch`만으로 상태 변경 — `useState` 직접 설정 금지
- SELECT_PIECE, DESELECT_PIECE, MOVE_PIECE 모두 gameReducer.ts에 이미 구현됨

**퍼포먼스**
- Board SVG는 정적 — 기물 선택/이동 시 Board 리렌더 없음 (React.memo 보호)
- HintOverlay: `useMemo`로 valid moves 계산 — selectedPiece/board 변경 시에만 재계산

**에러 처리 (FR10)**
- 잘못된 이동 → 조용히 무시 — gameReducer.ts MOVE_PIECE가 이미 처리
- DESELECT_PIECE는 항상 안전 — selectedPiece === null이어도 reducer가 조용히 처리

**Anti-patterns (절대 금지)**
- `Board.tsx` 수정 — 정적 컴포넌트, React.memo 보호
- `pieces/rules/cha.ts`에 React/DOM import
- `constants.ts` 외 매직 넘버 (560, 620, 22 등 — 상수에서 파생하거나 의미 있는 명칭 부여)
- `getValidMoves` 대신 `getChaValidMoves` 직접 호출
