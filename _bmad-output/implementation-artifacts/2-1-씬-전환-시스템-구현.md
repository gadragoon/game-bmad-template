# Story 2.1: 씬 전환 시스템 구현

Status: done

## Story

As a player,
I want the game to progress through 6 scenes in a fixed linear order,
so that I experience the complete narrative journey from opening to ending without getting lost.

## Acceptance Criteria

1. **Given** the game loads
   **When** the initial state is inspected
   **Then** `state.scene === 'opening'` and `state.phase === null`, 그리고 Opening 씬 컴포넌트가 렌더됨

2. **Given** any narrative scene is displayed
   **When** the player triggers "계속하기"
   **Then** `dispatch({ type: 'NEXT_SCENE' })` advances the scene in order: `opening → cha → ma → po → jol → ending`

3. **Given** any scene component is active
   **When** NEXT_SCENE fires
   **Then** the correct next scene component renders immediately — 중간 로딩 상태나 flash 없음

4. **Given** the game is on the ending scene
   **When** the scene is displayed
   **Then** "계속하기" 버튼이 없고 NEXT_SCENE을 dispatch해도 씬 변화 없음

5. **Given** any experience scene (`cha`, `ma`, `po`, `jol`)
   **When** NEXT_SCENE으로 해당 씬에 진입
   **Then** `state.phase === 'intro'` 자동 설정 (gameReducer가 처리)

### 현재 구현 상태 — 변경 없이 재사용되는 파일들

**`state/gameReducer.ts`** (이미 완전 구현됨, 변경 불필요):
```typescript
export const SCENE_ORDER: readonly Scene[] = ['opening', 'cha', 'ma', 'po', 'jol', 'ending'] as const
const NARRATIVE_SCENES = new Set<Scene>(['opening', 'ending'])

case 'NEXT_SCENE': {
  const idx = SCENE_ORDER.indexOf(state.scene)
  if (idx === SCENE_ORDER.length - 1) return state  // ending에서 무시 ✓
  const next = SCENE_ORDER[idx + 1]
  return {
    ...state,
    scene: next,
    phase: NARRATIVE_SCENES.has(next) ? null : 'intro',  // 체험 씬 → phase='intro' 자동 ✓
    selectedPiece: null,
  }
}
case 'SET_PHASE': return { ...state, phase: action.phase }
```

### App.tsx 구조 변경 (가장 중요한 변경)

```tsx
// BEFORE: Board/PiecesLayer를 직접 렌더
function App() {
  return (
    <GameProvider>
      <div className={styles.boardContainer}>
        <Board />
        <PiecesLayer />
      </div>
    </GameProvider>
  )
}

// AFTER: SceneRouter가 씬을 결정 (GameProvider 내부라서 useGameState 사용 가능)
function SceneRouter() {
  const { scene } = useGameState()
  switch (scene) {
    case 'opening': return <Opening />
    case 'cha':     return <ExperienceCha />
    case 'ma':      return <ExperienceMa />
    case 'po':      return <ExperiencePo />
    case 'jol':     return <ExperienceJol />
    case 'ending':  return <Ending />
    default:        return null
  }
}

function App() {
  return (
    <GameProvider>
      <SceneRouter />
    </GameProvider>
  )
}
```

**주의:** `SceneRouter`는 반드시 `GameProvider` 내부에 있어야 `useGameState`를 사용할 수 있음. App 함수 내에서 직접 `useGameState`를 호출하면 Provider 외부 에러 발생.

### 기존 Playwright 테스트 리그레션 수정 (필수)

**문제:** Story 2.1 이후, `page.goto('/')` 시 Opening 씬이 렌더됨. 보드는 보이지 않음.
기존 테스트들(`[aria-label="기물 po"]` 등)이 모두 실패함.

**수정 패턴 (story-1-6, 1-7, 1-8 모두 동일):**
```typescript
test.beforeEach(async ({ page }) => {
  consoleErrors = []
  page.on('console', msg => { if (msg.type() === 'error') consoleErrors.push(msg.text()) })
  await page.goto('/')
  // Story 2.1+: Opening 씬 통과 → ExperienceCha (보드 전체 접근 가능)
  await page.locator('button:has-text("계속하기")').click()
  await page.waitForSelector('[aria-label="기물 ma"]', { timeout: 5000 })  // 각 파일에 맞는 기물로
})
```

**왜 ExperienceCha에서 모든 기물이 보이는가:** 초기 board 상태에 cha/ma/po/jol 모두 배치됨. ExperienceCha 스텁은 Board + PiecesLayer를 렌더하므로 모든 기물 접근 가능.

### Story 2.1 Playwright 테스트 계획

```typescript
// tests/story-2-1-scene-transition.spec.ts
const SCENE_ORDER = ['opening', 'cha', 'ma', 'po', 'jol', 'ending'] as const

test('AC1: 초기 씬 opening 렌더 — 보드 미노출', async ({ page }) => {
  await page.goto('/')
  await expect(page.locator('[data-scene="opening"]')).toBeVisible()
  // Opening은 내러티브 씬 — 보드 없음
  await expect(page.locator('[aria-label="기물 cha"]')).not.toBeVisible()
})

test('AC2: 씬 순서 6단계 전체 순회', async ({ page }) => {
  await page.goto('/')
  for (let i = 0; i < SCENE_ORDER.length - 1; i++) {
    const current = SCENE_ORDER[i]
    const next = SCENE_ORDER[i + 1]
    await expect(page.locator(`[data-scene="${current}"]`)).toBeVisible()
    await page.locator(`[data-scene="${current}"] button:has-text("계속하기")`).click()
    await expect(page.locator(`[data-scene="${next}"]`)).toBeVisible()
  }
  // 최종: ending 도달 확인
  await expect(page.locator('[data-scene="ending"]')).toBeVisible()
})

test('AC3: 씬 전환 즉시 렌더 — flash 없음', async ({ page }) => {
  await page.goto('/')
  await page.locator('[data-scene="opening"] button:has-text("계속하기")').click()
  await expect(page.locator('[data-scene="cha"]')).toBeVisible()
  await expect(page.locator('[data-scene="opening"]')).not.toBeVisible()
})

test('AC4: ending에서 계속하기 없음, 씬 변화 없음', async ({ page }) => {
  await page.goto('/')
  // ending까지 이동
  for (const scene of ['opening', 'cha', 'ma', 'po', 'jol']) {
    await page.locator(`[data-scene="${scene}"] button:has-text("계속하기")`).click()
  }
  await expect(page.locator('[data-scene="ending"]')).toBeVisible()
  // 계속하기 버튼 없음
  await expect(page.locator('[data-scene="ending"] button:has-text("계속하기")')).not.toBeVisible()
})

test('AC5: 체험 씬 진입 시 phase=intro 자동 설정', async ({ page }) => {
  await page.goto('/')
  await page.locator('[data-scene="opening"] button:has-text("계속하기")').click()
  // NEXT_SCENE reducer: 체험 씬 진입 시 phase='intro' 자동 설정
  await expect(page.locator('[data-scene="cha"][data-phase="intro"]')).toBeVisible()
})
```
