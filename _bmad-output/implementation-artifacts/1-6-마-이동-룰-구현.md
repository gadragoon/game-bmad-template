# Story 1.6: 마(馬) 이동 룰 구현

Status: done

## Story

As a player,
I want to move the Ma piece using its L-shaped movement rule with adjacent-square blocking,
so that I experience the horse's distinctive movement pattern.

## Acceptance Criteria

1. **Given** the Ma piece on an unobstructed interior square
   **When** `getValidMoves('ma', pos, board)` is called
   **Then** it returns up to 8 L-shaped destinations (1 orthogonal step + 1 diagonal step in each of 4 directions)

2. **Given** a piece occupying a square orthogonally adjacent to the Ma
   **When** valid moves are calculated for the Ma
   **Then** the 2 diagonal destinations in that blocked direction are excluded (인접 칸 막힘 rule)

3. **Given** the Ma piece near a board edge
   **When** valid moves are calculated
   **Then** only in-bounds destinations are returned — no out-of-bounds positions

4. **Given** a valid Ma destination
   **When** the player selects the Ma and clicks that destination
   **Then** the Ma moves to that square

5. **Given** `pieces/rules/ma.ts`
   **When** the file is inspected
   **Then** it has zero React or DOM imports — pure function only

## Dev Notes

### 핵심 구현: getMaValidMoves 알고리즘

```
직교 방향 4개 × 대각선 2개 = 최대 8개 목적지

각 직교 방향(dr, dc)에 대해:
  인접 칸 = (pos.row + dr, pos.col + dc)
  
  if 인접 칸이 보드 안이고 board[인접칸.row][인접칸.col] !== null:
    → 이 방향은 막힘, 2개 대각선 모두 skip
  else:
    대각선 1 = (pos.row + dr*2 + dc, pos.col + dc*2 + dr)  ← 이 공식 틀리기 쉬움!
    대각선 2 = (pos.row + dr*2 - dc, pos.col + dc*2 - dr)
    
    각 대각선 목적지에 대해:
      in-bounds 체크
      목적지 비어있는지 체크 (단일 팀 — 기물 있으면 제외)
      통과 시 result에 추가
```

**대각선 방향 공식 (중요 — 틀리기 쉬운 부분):**

직교 방향이 `(dr, dc)`일 때, 대각선 2개 목적지의 row/col 오프셋:
- 직교 `(0, +1)` (우): 목적지 → `(-1, +2)`, `(+1, +2)` ✅
- 직교 `(0, -1)` (좌): 목적지 → `(-1, -2)`, `(+1, -2)` ✅
- 직교 `(-1, 0)` (상): 목적지 → `(-2, -1)`, `(-2, +1)` ✅
- 직교 `(+1, 0)` (하): 목적지 → `(+2, -1)`, `(+2, +1)` ✅

공식으로 표현: 직교 `(dr, dc)`에서 대각선 벡터 2개:
- `(dr*2 + dc, dc*2 + dr)` → 좌측 대각선
- `(dr*2 - dc, dc*2 - dr)` → 우측 대각선

(또는 각 방향을 명시적으로 `STEPS` 배열로 하드코딩하는 것도 가능 — 코드 가독성이 더 좋음)

**권장 구현 패턴:**

```typescript
import type { Position, Board } from '../../types'
import { BOARD } from '../../constants'

// 각 직교 방향과 그 방향에서의 2개 대각선 목적지 오프셋
const MA_STEPS: Array<{
  block: { dr: number; dc: number }       // 인접 칸 막힘 체크
  targets: Array<{ dr: number; dc: number }> // 목적지 2개
}> = [
  { block: { dr: -1, dc:  0 }, targets: [{ dr: -2, dc: -1 }, { dr: -2, dc: +1 }] }, // 상
  { block: { dr: +1, dc:  0 }, targets: [{ dr: +2, dc: -1 }, { dr: +2, dc: +1 }] }, // 하
  { block: { dr:  0, dc: -1 }, targets: [{ dr: -1, dc: -2 }, { dr: +1, dc: -2 }] }, // 좌
  { block: { dr:  0, dc: +1 }, targets: [{ dr: -1, dc: +2 }, { dr: +1, dc: +2 }] }, // 우
]

export function getMaValidMoves(pos: Position, board: Board): Position[] {
  const result: Position[] = []
  for (const { block, targets } of MA_STEPS) {
    const br = pos.row + block.dr
    const bc = pos.col + block.dc
    // 인접 칸이 보드 안에 있고 기물이 있으면 → 막힘, skip
    if (br >= 0 && br < BOARD.rows && bc >= 0 && bc < BOARD.cols && board[br][bc] !== null) {
      continue
    }
    for (const { dr, dc } of targets) {
      const tr = pos.row + dr
      const tc = pos.col + dc
      if (tr < 0 || tr >= BOARD.rows || tc < 0 || tc >= BOARD.cols) continue
      if (board[tr][tc] !== null) continue  // 목적지 기물 있음 → 단일 팀, 포획 불가
      result.push({ row: tr, col: tc })
    }
  }
  return result
}
```

> **board 인덱싱:** 반드시 `board[row][col]` 순서. `board[col][row]` 실수 금지.

### Project Context Rules

**장기 룰 엔진 (핵심)**
- `getValidMoves(pieceId, pos, board)` 단일 진입점 — `getMaValidMoves` 직접 호출 금지
- 룰 함수는 **순수 함수만** — React/DOM import 절대 금지, 사이드이펙트 없음
- `BOARD` import from `../../constants` — `board.length`, 숫자 리터럴 9/10 금지

**단일 팀 POC 시나리오**
- 기물 포획 없음 — 목적지에 기물이 있으면 결과에서 제외 (덮어쓰기 방지)
- 차(車) 룰: 기물이 있는 칸은 포함 안 함 — 마(馬)도 동일 원칙 적용

**퍼포먼스**
- `getMaValidMoves`는 순수 함수 — PiecesLayer의 `useMemo` deps에 의해 캐싱됨
- 불필요한 연산 없음 — 최대 4방향 × 2목적지 = 8번 체크로 O(1) 성능

**에러 처리**
- 잘못된 이동 → `gameReducer.ts`의 MOVE_PIECE가 `getValidMoves` 재검증으로 자동 차단
- 예외를 UI에 노출하지 않음

**Anti-patterns (절대 금지)**
- `ma.ts`에 React/DOM import
- `board.length` 또는 숫자 리터럴 9, 10 — `BOARD.rows`, `BOARD.cols` 사용
- 인접 칸 막힘 조건 누락 — 단순 8개 목적지만 반환하면 AC 2 실패
- 목적지 기물 체크 누락 — gameReducer 덮어쓰기 버그 유발
