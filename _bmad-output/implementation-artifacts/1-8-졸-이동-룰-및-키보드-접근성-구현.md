# Story 1.8: 졸(卒) 이동 룰 및 키보드 접근성 구현

Status: done

## Story

As a player,
I want to move the Jol piece forward or sideways and use keyboard navigation to play,
so that I experience the soldier's constrained movement and can play without a mouse.

## Acceptance Criteria

1. **Given** the Jol piece on any interior square
   **When** `getValidMoves('jol', pos, board)` is called
   **Then** it returns only the square directly forward and the squares directly left and right (maximum 3 squares)

2. **Given** the Jol piece at the forward-most edge (row:9, 보드 하단)
   **When** valid moves are calculated
   **Then** the forward square is not returned — Jol cannot retreat or advance past this edge (후퇴 불가)

3. **Given** the Jol piece at a side edge (col:0 또는 col:8)
   **When** valid moves are calculated
   **Then** only in-bounds lateral and forward moves are returned

4. **Given** the board is rendered in the browser
   **When** a player presses Tab
   **Then** keyboard focus cycles through interactive elements (selectable pieces and, when selected, valid destination squares)

5. **Given** a piece or valid square has keyboard focus
   **When** the player presses Enter or Space
   **Then** the same action fires as a mouse click — selection, move, or deselection (FR11)

6. **Given** `pieces/rules/jol.ts`
   **When** the file is inspected
   **Then** it has zero React or DOM imports — pure function only

## Dev Notes

### 보드 초기 배치 (constants.ts 기준)

```
cha: (col:0, row:5)   — board[5][0]
ma:  (col:2, row:7)   — board[7][2]
po:  (col:4, row:2)   — board[2][4]
jol: (col:4, row:5)   — board[5][4]   ← 이 스토리 주인공
```

**좌표계:**
- `col: 0–8` (왼쪽→오른쪽), `row: 0–9` (위→아래)
- 졸의 **전진** = 아래 방향 = row 증가 (row+1)
- 졸의 **후퇴** = 위 방향 = row 감소 (row-1) → 절대 미포함
- 졸의 **좌** = col 감소 (col-1), **우** = col 증가 (col+1)

### 핵심 알고리즘: getJolValidMoves

마(馬)/포(砲)보다 훨씬 단순한 candidates filter 패턴:

```typescript
import type { Position, Board } from '../../types'
import { BOARD } from '../../constants'

export function getJolValidMoves(pos: Position, board: Board): Position[] {
  const candidates = [
    { row: pos.row + 1, col: pos.col },     // 전진 (후퇴 불가 — row-1 없음)
    { row: pos.row,     col: pos.col - 1 }, // 좌
    { row: pos.row,     col: pos.col + 1 }, // 우
  ]
  return candidates.filter(({ row, col }) =>
    row >= 0 && row < BOARD.rows &&
    col >= 0 && col < BOARD.cols &&
    board[row][col] === null  // 단일 팀 POC — 기물 있으면 제외
  )
}
```

**졸 알고리즘 특성:**
- candidates 배열에 `{row-1, col}` (후퇴) 없음 → AC2 자동 충족
- OOB filter → AC3 자동 충족
- `board[row][col] === null` → Story 1.5 deferred "MOVE_PIECE 덮어쓰기 위험" 해소
